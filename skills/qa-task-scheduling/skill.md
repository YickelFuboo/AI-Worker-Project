# 任务调度

## 功能描述

根据工作流定义文件动态构建任务图，调度各 Agent 按依赖顺序执行。维护全局上下文，传递阶段产物路径，监控执行状态。

## 所属 Agent

流程编排 Agent

## 适用场景

- 正向需求交付全流程启动
- 存量反向分析流程启动
- 需要按工作流定义调度多个 Agent 协同工作
- 需要维护跨 Agent 的全局上下文

## 工作方式

### 执行步骤

1. **读取工作流定义**：加载 `.agent/workflow_definitions.yaml`，解析工作流模板
2. **构建任务图**：根据工作流的 `steps` 定义和 `depends_on` 关系，构建有向无环图（DAG）
3. **上下文初始化**：初始化全局上下文对象，包含工作空间路径、需求 ID、产物路径映射等
4. **调度执行**：按拓扑顺序调度 Agent 执行，将上游 Agent 的输出路径传递给下游 Agent
5. **状态监控**：记录每个步骤的执行状态（pending/running/completed/failed），更新执行日志
6. **完成通知**：全流程执行完成后，发送通知或回调

### 注意事项

- 任务调度必须严格按 DAG 依赖顺序执行，无依赖的步骤可并行
- 每个步骤执行前检查上游步骤是否全部成功完成
- 全局上下文应包含所有步骤的输入输出路径，便于下游 Agent 定位产物
- 调度过程中记录执行日志到 `.agent/` 目录

### 输出要求

- 输出执行状态记录：步骤ID、Agent名称、开始时间、结束时间、状态
- 维护全局上下文（可序列化为 YAML/JSON，供下游 Agent 读取）
- 异常时输出中断位置和上下文快照
