# 变更范围细化

## 功能描述

将架构变更细化为具体的模块、文件、函数级变更清单。从架构影响域分析结果出发，定位到代码仓中的具体模块和文件，生成精确的变更范围清单，为编码实现提供明确的修改目标。

## 所属 Agent

实现设计 Agent

## 适用场景

- 逻辑流设计完成后，需要确定具体修改哪些文件和函数
- 需要评估代码变更的工作量和影响范围
- 需要生成 `repo_changes/overview.md` 总览文档

## 工作方式

### 执行步骤

1. **读取架构变更说明**：理解需要变更的架构元素和接口
2. **读取代码仓设计**：加载各代码仓的 `spec.md`、`design.md` 及模块设计文档
3. **读取代码**：使用代码阅读工具定位涉及的文件和函数
4. **变更范围映射**：将架构变更映射到具体的文件路径、函数签名、数据结构
5. **影响面评估**：分析每个文件变更对上下游模块的影响
6. **生成总览文档**：输出 `repo_changes/overview.md`，汇总所有受影响代码仓和模块

### 注意事项

- 变更清单必须精确到文件路径和函数级别
- 需要区分新增、修改、删除三种变更类型
- 标注每个变更项的优先级和依赖关系
- 参考各代码仓 `.agent/rules/` 目录下的约束文件（`_index.md` 列出全部文件，按需加载 `约束/*.md`），遵守仓级约束

### 输出要求

- 总览输出：`requirements/{需求ID}/repo_changes/overview.md`，严格按模板格式：`templates/repo_changes_overview_template.md`
- 各仓细化：`requirements/{需求ID}/repo_changes/{仓名}/implementation_design.md`
- 包含文件级变更清单、函数签名变更、数据结构变更
