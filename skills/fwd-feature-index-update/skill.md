# 特性索引维护

## 功能描述

在正向流程中新增/修改/删除子特性或场景时，维护 `features/features_tree.yaml` 和各一级特性下的二级索引文件（`*_feature.yaml`）。确保特性树索引与实际特性文件保持一致，避免索引漂移。

## 所属 Agent

需求分析 Agent

## 适用场景

- 正向流程中新增子特性或场景
- 修改已有子特性的目录结构或场景列表
- 删除废弃的子特性或场景
- 业务变更说明生成后，需要同步更新特性索引

## 工作方式

### 执行步骤

1. **读取业务变更说明**：加载 `feature_changes/` 下的变更文件，识别新增/修改/删除的场景和子特性
2. **读取当前索引**：加载 `features/features_tree.yaml` 和相关一级特性的 `*_feature.yaml`
3. **比对差异**：识别索引与实际文件的不一致（新增文件未入索引、索引项文件不存在等）
4. **更新索引**：
   - 新增子特性/场景：在对应二级索引中追加条目
   - 修改：更新条目的 `name`、`spec`、`arch_ref`、`scenarios` 字段
   - 删除：从索引中移除条目
5. **校验引用完整性**：确保索引中每条 `file` 字段指向的文件实际存在
6. **更新时间戳**：更新索引文件的 `last_updated` 字段

### 注意事项

- 索引维护是正向流程的关键保障，索引漂移会导致后续 Agent 无法定位场景文件
- 删除场景前必须确认无其他子特性/需求引用
- 新增一级特性时需同步更新 `features_tree.yaml`
- 索引更新失败应阻断流程，避免下游 Agent 基于错误索引工作

### 输出要求

- 更新 `features/features_tree.yaml`（如有一级特性变更）
- 更新相关一级特性的 `*_feature.yaml`
- 输出索引变更摘要：新增/修改/删除的索引项清单
