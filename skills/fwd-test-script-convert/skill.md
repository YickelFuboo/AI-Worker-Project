# 测试脚本转换

## 功能描述

将测试用例 Markdown 文档转换为可执行的自动化测试脚本。使用项目测试框架（如 pytest、GTest）生成测试代码，包含测试环境初始化、步骤执行、断言验证和清理。

## 所属 Agent

测试开发 Agent

## 适用场景

- 测试用例文档完成后，需要自动化执行
- 需要将手工测试步骤转换为可重复执行的脚本
- 需要集成到 CI/CD 流水线中

## 工作方式

### 执行步骤

1. **读取测试用例**：加载 `tests/test_cases/` 下的测试用例 Markdown 文档
2. **读取脚本模板**：参考 `tests/scripts/` 中已有的测试脚本，提取框架使用模式
3. **读取部署拓扑**：加载 `architectures/deployment_view/deployment.md`，理解测试环境拓扑
4. **生成测试脚本**：将每个测试用例的步骤转换为测试函数，前置条件转换为 setUp，步骤转换为操作+断言
5. **生成测试数据**：如需要，生成测试用的输入数据和配置文件

### 注意事项

- 测试脚本必须支持独立运行和批量运行
- 每个测试函数必须包含清理逻辑，确保测试环境可复用
- 测试脚本应输出结构化日志（JSON 或 JUnit XML），便于测试分析 Agent 解析
- 参考 `tests/scripts/` 中已有脚本的编码风格

### 输出要求

- 输出路径：`tests/scripts/test_{特性名}.py`（或其他语言对应文件）
- 每个测试函数命名包含关联的测试用例 ID
- 包含 setUp/tearDown 或 fixture 定义
