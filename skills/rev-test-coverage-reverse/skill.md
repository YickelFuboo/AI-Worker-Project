# 测试覆盖逆向挖掘

## 功能描述

从现有测试脚本和测试用例反向关联覆盖的业务场景与架构元素。生成用例-场景-架构元素的覆盖映射，识别测试盲区。

## 所属 Agent

测试逆向 Agent

## 适用场景

- 存量项目有测试脚本但缺少覆盖映射
- 需要评估现有测试对业务场景和架构元素的覆盖情况
- 需要识别测试盲区以制定补充测试计划

## 工作方式

### 执行步骤

1. **读取测试脚本**：加载 `tests/scripts/` 下的所有自动化测试脚本
2. **读取测试用例**：加载 `tests/test_cases/` 下的测试用例文档
3. **读取业务场景**：加载 `features/` 下的所有场景交互流程文件
4. **读取架构元素**：加载 `architectures/logic_view/elements_tree.yaml` 和元素规格
5. **建立映射**：
   - 分析测试脚本中的函数名、注释、断言信息，匹配业务场景
   - 分析测试中调用的接口和模块，匹配架构元素
   - 生成 用例→场景→架构元素 的映射关系
6. **覆盖分析**：统计每个场景和架构元素的测试覆盖情况，识别未覆盖或覆盖不足的区域
7. **更新缓存**：更新 `.agent/cross_perspective_cache.yaml` 中的映射关系

### 注意事项

- 映射关系基于代码分析，置信度不足时应标注
- 覆盖分析需区分直接覆盖和间接覆盖
- 识别到的测试盲区应标注风险等级
- 参考 `tests/test_strategy.md` 中的覆盖策略

### 输出要求

- 输出覆盖映射文件（`tests/coverage_mapping.yaml`）：场景ID→测试用例ID→测试脚本→覆盖的架构元素
- 更新 `.agent/cross_perspective_cache.yaml`
- 输出覆盖分析报告：覆盖率统计、测试盲区清单、建议补充的测试场景
