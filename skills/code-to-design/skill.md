# 代码反向生成设计文档

## 功能描述

对存量代码仓（无设计文档的项目），基于已有代码反向生成技术设计文档。通过分析模块结构、类/函数关系、调用链、数据流向，还原出架构设计和模块详细设计。

## 适用场景

- 存量产品需要补齐设计文档
- 代码重构前需要理解现有设计
- 反向流程：代码 → Spec → **设计文档**
- 对应设计文档中的"存量产品反向生成设计"

## 工作方式

### 执行步骤

1. **前提检查**：确认已执行 `code-to-spec` 或已有 Spec 文档
2. **读取 Spec 文档**：了解功能划分和规格指标
3. **分析代码架构**：
   - 调用 `read-project-struct` 获取模块树
   - 识别模块间的依赖关系（通过 IMPORTS 边）
4. **分析模块内部设计**：
   - 对每个模块的核心文件，调用 `read-file-summary` 获取类/函数清单
   - 识别核心类、核心函数、数据结构
5. **分析接口关系**：
   - 通过 CALLS 边还原模块间接口调用关系
   - 生成模块间接口表
6. **分析关键流程**：
   - 选择 2-3 个核心业务场景
   - 追踪调用链，绘制 PlantUML 序列图
7. **分析 DFX 设计**：
   - 从代码中识别错误处理、重试、缓存、日志等模式
   - 还原性能/可靠性/安全设计手段
8. **编写设计文档**：
   - 按设计模板组织输出
   - 标注"由代码反推，待人工确认"的项

### 注意事项

- 设计文档包含设计理由（"为什么这么做"），代码反推只能得到"做了什么"，设计理由标注"待补充"
- 状态机从代码中还原可能不完整，需人工补充
- 调用链分析应限制深度（建议 ≤ 3 跳），避免输出膨胀
- 模块间接口表应注明"由 CALLS 边反推"

### 输出要求

- 仓级设计：`architectures/repo_design.md`
- 模块级设计：`architectures/modules/{module_name}/design.md`
- 所有反推条目标注 `[INFERRED]` 标记
- 待确认项列表
