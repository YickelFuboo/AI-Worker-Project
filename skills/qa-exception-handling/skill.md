# 异常处理

## 功能描述

处理工作流执行过程中的异常情况。根据工作流定义中的 `on_failure` 策略，执行相应的异常处理动作：暂停等待人工介入、回滚到指定步骤、拒绝并通知等。

## 所属 Agent

流程编排 Agent

## 适用场景

- 某个 Agent 执行失败时
- 阶段门禁不通过时
- 需要回滚到上游步骤重新执行时
- 需要人工介入决策时

## 工作方式

### 执行步骤

1. **捕获异常**：接收失败步骤的错误信息和上下文
2. **读取失败策略**：从工作流定义中查找该步骤的 `on_failure` 配置
3. **执行异常处理**：
   - `pause_for_human`：暂停工作流，生成上下文摘要，通知人工审核介入
   - `rollback_to:<step_id>`：回滚到指定步骤，清理下游产物，重新执行
   - `reject_and_notify`：终止工作流，生成失败报告，通知相关人员
4. **记录异常**：将异常详情写入变更日志和执行状态记录
5. **上下文保存**：保存失败时的上下文快照，便于人工排查

### 注意事项

- 异常处理策略在工作流定义中预先配置，不可临时更改
- `rollback_to` 时必须清理回滚点之后的所有产物，确保状态一致
- `pause_for_human` 时生成的上下文摘要必须包含足够的决策信息

### 输出要求

- 输出异常处理记录：异常类型、失败步骤、处理策略、处理结果
- 输出上下文摘要（供人工审核使用）
- 输出失败报告（`reject_and_notify` 时）
