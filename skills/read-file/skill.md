# 读取完整文件（兜底 L\* Tool）

## 功能描述

读取完整文件的原始源码。**仅当** L2 文件框架不足以判断时使用（如全局常量、复杂导入、宏定义等场景），Token 成本高，不作为默认读码路径。

## 适用场景

- L2 `read-file-summary` 返回的框架信息不足以确定改码点
- 需要查看文件级别的全局常量、类型别名、宏定义
- 需要理解复杂的文件级导入关系
- 文件包含大量顶层代码（非类/函数内）
- 对应设计文档 REQ-CG-D08：read_file 兜底

## 工作方式

### 执行步骤

1. **前提检查**：确认已执行过 L1、L2，且框架信息确实不足以判断
2. **接收参数**：project_id + 文件路径
3. **返回完整文件**：文件的完整原始源码
4. **压缩历史**（演进）：读取完成后，对话历史中的全文内容宜压缩为"已成功读取"摘要

### 注意事项

- **非默认路径**：禁止跳过 L1/L2 直接调用此 Tool
- Token 消耗大，仅在必要时使用
- 大文件（> 500 行）宜分段读取
- 读取后应在对话历史中尽早压缩，避免长期占用上下文
- 如果图谱未构建，此 Tool 退化为普通文件读取

### 停止使用条件

以下情况应立即停止继续读取更多文件，回到分层策略：
- 已通过 L1/L2 定位到明确改码点
- 当前文件与需求无关
- 已累计读取超过 3 个完整文件（Token 预算告警）

## 输出要求

```
文件: {file_path}
语言: {language}
行数: {n}

---
{完整文件源码}
---
```
