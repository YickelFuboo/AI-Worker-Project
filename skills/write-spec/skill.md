# 编写功能规格说明书

## 功能描述

将需求分析文档转化为功能规格说明书（Spec）。定义功能的精确行为、接口契约、数据结构和验收标准。Spec 是设计与实现的唯一事实来源。

## 适用场景

- 需求分析完成后，需要编写精确的技术规格
- 需要定义 API 接口的请求/响应格式
- 需要将业务需求翻译为技术规格指标
- 正向流程：需求分析 → **规格说明书** → 设计文档 → 代码

## 工作方式

### 执行步骤

1. **读取需求文档**：从 `requirements/` 目录读取对应的需求分析文档
2. **读取现有 Spec**：如已有仓级 Spec（`repo_spec.md`），先阅读了解现有规格
3. **提取功能规格**：从需求中提取功能规格项，定义精确的指标与目标值
4. **定义接口契约**：如有 API 变更，定义请求/响应格式、错误码
5. **定义数据结构**：明确新增或变更的数据实体、字段、校验规则
6. **编写验收标准**：以 Given-When-Then 格式定义可测试的验收条件
7. **输出 Spec 文档**：按模板输出到指定位置

### 注意事项

- Spec 是技术契约，措辞必须精确，避免"大概"、"可能"等模糊词
- 接口变更必须兼容性评估：是否破坏现有调用方
- 数据字段必须明确类型、是否必填、校验规则
- 如有不确定的规格项，标注"待确认"并说明影响
- 参考现有 `knowledge/模板库/` 中的模板结构

### 输出要求

- 文档路径：`requirements/specs/{feature_name}_spec.md`
- 如为仓级规格更新，同步更新 `repo_spec.md`
- 包含完整的功能规格指标表、数据结构定义、验收标准
