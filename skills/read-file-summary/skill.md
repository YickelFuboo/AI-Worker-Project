# 读取文件框架（L2 读码 Tool）

## 功能描述

获取单个文件的框架视图：文件职责摘要 + 类列表 + 函数/方法列表（各带一行说明），**故意不包含实现代码**。用于在候选文件中锁定类或函数名，是分层读码策略的第二层（L2）。

## 适用场景

- L1 已锁定候选模块/目录，需要进一步缩小到具体文件
- 需要知道某个文件里有哪些类、哪些函数
- 需要在多个候选函数中选择最相关的一个
- 对应设计文档中的 `read_file_summary` Tool

## 工作方式

### 执行步骤

1. **接收参数**：project_id + 文件路径（相对于项目根）
2. **查询图谱框架视图**：从图存储读取指定 File 节点的子符号列表
3. **返回结构化结果**：
   - 文件摘要（自然语言职责说明）
   - 类列表（类名 + 一行职责摘要）
   - 函数/方法列表（函数名 + 签名 + 一行职责摘要）
   - 导入列表（本文件导入了哪些模块）
4. **返回格式**：结构化文本

### 注意事项

- **绝对不返回**函数体、方法实现等源码内容
- 如果文件较大，符号列表可能很长，可先返回类列表，Agent 再指定查看某个类的方法
- 如果图谱未构建或文件不在图谱中，提示先执行 build-graph
- 摘要的一行说明应足够区分各符号的职责，避免千篇一律

### 输出要求

返回结构：
```
文件: {path}
语言: {language}
摘要: {文件职责的一句话说明}

导入:
  - {import_1}
  - {import_2}

类:
  - {ClassName} — {类职责摘要}
    ├── {method_1}({params}) — {方法职责摘要}
    └── {method_2}({params}) — {方法职责摘要}

函数:
  - {func_1}({params}) — {函数职责摘要}
  - {func_2}({params}) — {函数职责摘要}
```
