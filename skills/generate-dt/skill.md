# 生成 DT 测试用例

## 功能描述

基于功能规格说明书（Spec）或接口定义，自动生成 DT（Development Testing）测试用例。覆盖正常场景、边界值、异常输入和错误码，以 Given-When-Then 格式输出可执行的测试用例。

## 适用场景

- 接口规格定义完成后，需要生成测试用例
- Spec-Driven 开发流程的阶段二：接口规格 → 测试用例
- 回归测试用例补充
- 对应参考规则中的 `generate-dt`

## 工作方式

### 执行步骤

1. **读取接口规格**：从 Spec 文档中提取接口的 Request/Response 定义、字段校验规则、错误码列表
2. **生成正常场景用例**：
   - 为每个接口生成至少 1 个正常调用用例
   - 覆盖必填字段全部提供、可选字段部分提供
3. **生成边界值用例**：
   - 字段最小/最大值
   - 空字符串、null、空数组
   - 字段长度边界（等于、少 1、多 1）
4. **生成异常用例**：
   - 缺少必填字段
   - 字段类型错误
   - 字段值超出范围
   - 每个错误码至少 1 个用例
5. **生成组合用例**（如适用）：
   - 多个可选字段的排列组合
   - 接口调用顺序依赖
6. **输出测试用例文档**：按 Given-When-Then 格式组织

### 注意事项

- 测试用例必须可独立执行，不依赖其他用例的执行顺序
- Given 中描述的前置条件必须是可构造的
- Then 中的断言必须是可验证的（不依赖主观判断）
- 如果接口规格不完整（字段缺少校验规则），标注该用例为"待补充"
- 输出位置：`tests/{feature_name}_dt_cases.md`

### 输出要求

```
DT 测试用例: {feature_name}

接口: {Method} {Path}

正常场景:
  TC-001:
    Given: {前置条件}
    When: {触发动作，含具体请求数据}
    Then: {预期结果，含具体响应断言}

边界场景:
  TC-002: ...

异常场景:
  TC-003: ...
```
