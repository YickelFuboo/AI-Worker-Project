# 读取函数/类源码（L3 读码 Tool）

## 功能描述

获取指定函数或类的完整实现源码（含签名、注释、源码体）。用于最终确认改码点并理解实现细节，是分层读码策略的第三层（L3）。

## 适用场景

- L2 已锁定候选函数/类，需要查看完整实现确认改码位置
- 需要理解函数的详细实现逻辑
- 需要确认函数签名、参数类型、返回值
- 对应设计文档中的 `read_function_code` 和 `read_class_code` Tool

## 工作方式

### 执行步骤

1. **接收参数**：project_id + 文件路径 + 符号名（函数名或类名）
2. **查询图谱源码视图**：从图存储读取指定符号节点的源码属性
3. **返回完整源码**：符号的完整源码文本
4. **附带上下文**（可选）：该符号的调用方列表、被调方列表（用于影响分析）

### 注意事项

- L3 仅应在 L1→L2 已锁定符号后使用，**禁止**作为默认读码方式
- 返回的源码可能较长，Agent 应聚焦于改码相关部分
- 如果符号有多个重载，返回全部重载的签名和位置
- 类级别查询返回类的完整定义（含所有方法）
- 函数级别查询仅返回该函数的实现

### 输出要求

返回结构：
```
符号: {class_name|function_name}
类型: 类/函数
文件: {file_path}
签名: {full_signature}

源码:
```
{完整源码内容}
```

调用方（可选）:
  - {caller_1}
  - {caller_2}

被调方（可选）:
  - {callee_1}
```
