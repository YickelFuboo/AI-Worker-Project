# 图谱增量更新

## 功能描述

当源码发生变更后，对变更文件或目录执行增量更新，替换图谱中对应子树，保证图谱与当前仓库一致，避免全量重建的高昂成本。

## 适用场景

- 开发者保存文件后触发增量刷新
- CI 流水线检测到代码变更
- 重构某个模块后需要同步图谱
- 子功能二 REQ-CG-A05 的实现

## 工作方式

### 执行步骤

1. **识别变更范围**：通过 git diff 或文件系统监听，确定变更的文件和目录列表
2. **删除旧子图**：将变更文件对应的旧节点（File 及下属 Class/Function）从图谱中移除
3. **分析变更文件**：对每个变更文件重新执行静态分析，提取符号与依赖
4. **生成新摘要**：为变更的符号重新生成语义摘要
5. **写入新子图**：将新节点与边写入图谱，连接回父目录节点
6. **级联刷新**：如果函数签名变更，刷新调用该函数的相关边
7. **聚合更新**：沿目录树向上重新聚合受影响目录的模块摘要
8. **输出版本记录**：更新图谱版本号与更新时间

### 注意事项

- "删旧写新"应在事务内完成，保证一致性
- 增量更新后图谱版本号必须递增，供 Agent 判断图谱是否过期
- 如果变更涉及大量文件（>50%），建议全量重建而非增量更新
- 删除文件时，同时清理孤立的调用边（指向已删除符号的 CALLS）
- 目录摘要需要沿路径向上逐级刷新

### 输出要求

- 增量更新报告：`architectures/graph_update_log.md`
- 包含：更新时间、变更文件数、新增/更新/删除节点数、版本号
