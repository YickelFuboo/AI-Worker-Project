# 加固建议生成

## 功能描述

基于质量度量结果和测试分析，生成质量加固建议。识别测试薄弱区域，给出补充测试建议、代码加固建议和流程改进建议。

## 所属 Agent

测试分析 Agent

## 适用场景

- 质量度量完成后，发现覆盖缺口或高风险区域
- 需要给出针对性的质量改进建议
- 需要为后续迭代提供质量提升方向

## 工作方式

### 执行步骤

1. **读取质量报告**：加载质量度量结果和失败分析报告
2. **读取覆盖缺口**：识别未覆盖或覆盖不足的场景和代码路径
3. **读取架构风险**：分析高风险架构元素（如核心路径、共享状态、并发区域）
4. **生成建议**：按优先级输出加固建议列表，包括：
   - 补充测试建议（场景、用例）
   - 代码加固建议（错误处理、边界检查、超时机制）
   - 流程改进建议（质量门禁调整、规范补充）
5. **优先级排序**：按风险等级和影响范围排序

### 注意事项

- 加固建议必须具体、可执行，不提供笼统的"加强测试"类建议
- 高风险区域的加固建议应标注为必须执行
- 参考 `knowledge/历史决策/` 中的类似场景处理经验

### 输出要求

- 输出加固建议报告：建议列表（优先级、类别、具体措施、预期效果）
- 标注必须执行的高优先级项
- 附建议的执行工作量评估
