# 架构影响域分析

## 功能描述

根据需求分析结果和业务变更说明，分析架构层面的影响范围。确定哪些架构元素（服务、组件、子系统）和接口需要变更，生成架构元素变更说明。

## 所属 Agent

架构设计 Agent

## 适用场景

- 业务变更说明完成后，需要确定架构影响范围
- 需要评估需求变更涉及哪些架构元素及其接口
- 为后续实现设计提供架构侧的变更输入

## 工作方式

### 执行步骤

1. **读取需求文档**：理解 `requirement.md` 中的功能范围和场景列表
2. **读取元素树**：加载 `architectures/logic_view/elements_tree.yaml`，获取全量架构元素和依赖关系
3. **读取元素规格**：阅读受影响元素的 `spec.md` 和 `interfaces.yaml`
4. **读取架构关联**：加载相关子特性的 `arch_ref.yaml`，建立场景→架构元素的映射
5. **影响域分析**：逐元素判断是否需要变更（新增/修改/删除），分析每个接口的变更需求
6. **生成变更说明**：按模板输出到 `requirements/{需求ID}/architecture_changes/{元素名}/` 下

### 注意事项

- 架构变更必须考虑对下游元素和外部系统的影响
- 接口变更需评估向后兼容性，破坏性变更必须特别标注
- 参考 `knowledge/模板库/架构元素设计模板.md` 的格式
- 参考 `knowledge/历史决策/` 中的设计决策记录，避免重复设计错误

### 输出要求

- 输出路径：`requirements/{需求ID}/architecture_changes/{元素名}/AR-{序号}-{元素名}_变更说明.md`
- 严格按模板格式输出：`templates/arch_change_template.md`
- 包含受影响的架构元素列表、接口变更细节、依赖关系变更
- YAML 元数据头部包含需求ID、元素ID、变更类型
