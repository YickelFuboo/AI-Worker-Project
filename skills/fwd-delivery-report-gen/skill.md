# 交付报告生成

## 功能描述

汇总需求迭代的全流程产物，生成最终的交付报告 `delivery_report.md`。报告涵盖需求到验证的完整闭环：需求范围、业务/架构/实现/测试各视角变更摘要、质量度量结果、遗留风险与建议，作为需求交付的最终归档文档。

## 所属 Agent

测试分析 Agent（闭环阶段，也可由流程编排 Agent 触发）

## 适用场景

- 正向需求交付全流程的最后一个步骤
- 需要生成需求迭代闭环总结
- 需要为版本发布提供交付归档文档
- 需要向 stakeholder 汇报需求交付结果

## 工作方式

### 执行步骤

1. **汇总各视角产物**：
   - 业务：`requirements/{需求ID}/需求分析.md`、`feature_changes/`
   - 架构：`architecture_changes/`
   - 实现：`repo_changes/overview.md` 及各仓 `implementation_design.md`
   - 验收：`test_changes/`、测试执行报告、质量度量报告
2. **读取质量报告**：加载 `quality-metrics` 和 `hardening-suggestions` 的输出
3. **读取失败分析**：加载 `failure-root-cause` 的失败分类与根因
4. **读取继承性报告**：评估兼容性风险是否已闭环
5. **生成交付报告**：按模板输出 `delivery_report.md`，包含：
   - 需求概述与范围
   - 各视角变更摘要（含统计：新增/修改/删除文件数、场景数、用例数）
   - 质量度量结果（覆盖率、通过率、缺陷数）
   - 兼容性风险评估结论
   - 遗留问题与建议
   - 关联 commit 列表与版本包信息

### 注意事项

- 交付报告是流程闭环的最终产物，必须如实反映质量状况，不夸大不隐瞒
- 遗留风险必须明确标注是否阻断发布
- 关联 commit 哈希和版本包签名必须准确，便于追溯
- 参考 `knowledge/模板库/交付报告模板.md` 的格式
- 若质量门禁未通过，应在报告中明确标注"未达交付标准"

### 输出要求

- 输出路径：`requirements/{需求ID}/delivery_report.md`
- 严格按模板格式输出：`templates/delivery_report_template.md`
- 包含需求概述、各视角变更摘要、质量度量、风险结论、遗留建议
- YAML 元数据头部包含需求ID、版本号、生成时间、交付结论（通过/有条件通过/未通过）
