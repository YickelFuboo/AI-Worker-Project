# 阶段门禁判定

## 功能描述

在各研发阶段完成后，依据阶段门禁规则执行质量检查，判定该阶段产物是否可以进入下一阶段。门禁不通过时阻断流程或生成告警，确保每个阶段的产出物达到质量基线。

## 所属 Agent

质量质检 Agent

## 适用场景

- 需求分析、架构设计、实现设计、编码、测试各阶段完成后
- 需要判定阶段产物是否满足门禁条件
- 需要自动阻断不达标产物进入下游

## 工作方式

### 执行步骤

1. **读取阶段产物**：加载当前阶段产出的所有文件（如 `requirement.md`、`architecture_changes/` 等）
2. **读取门禁规则**：加载 `knowledge/质量门禁/` 下对应阶段的 YAML 门禁规则：
   - 需求阶段门禁.yaml
   - 架构阶段门禁.yaml
   - 实现阶段门禁.yaml
   - 代码阶段门禁.yaml
   - 测试阶段门禁.yaml
3. **读取一致性规则**：加载 `.agent/consistency_rules.yaml` 中的跨视角约束
4. **执行检查**：逐条执行门禁规则中的检查项，判定通过/不通过/告警
5. **门禁判定**：
   - 所有 error 级别规则通过 → 门禁通过
   - 存在 error 级别规则不通过 → 门禁不通过，阻断流程
   - 仅 warning 级别规则不通过 → 门禁通过但生成告警清单
6. **生成质检报告**：输出判定结果和缺陷清单

### 注意事项

- 门禁判定是硬约束，不通过不得进入下一阶段
- error 级别规则的不通过项必须包含具体的缺陷描述和修复建议
- warning 级别的不通过项记录到告警清单，可在人工审核后放行
- 质检报告应可追溯到具体的规则 ID

### 输出要求

- 输出质检报告，严格按模板格式：`templates/gate_check_report_template.md`
- 包含门禁结论（通过/不通过/告警通过）、检查项统计、缺陷详情
- 缺陷详情包含：规则 ID、严重级别、缺陷描述、修复建议
