# 产物自动校验

## 功能描述

对阶段产物进行自动化校验，包括跨视角一致性校验和产物内在质量校验。检查不同视角产物之间的引用完整性（如场景引用的接口是否存在、测试用例是否覆盖声明的场景），以及产物本身的结构完整性和格式正确性。

## 所属 Agent

质量质检 Agent

## 适用场景

- 阶段产物生成后，需要校验与上游视角产物的一致性
- 需要检查产物文件的格式、结构、引用完整性
- 需要发现跨视角的信息不一致（场景 ↔ 接口 ↔ 代码 ↔ 测试）

## 工作方式

### 执行步骤

1. **读取阶段产物**：加载当前阶段产出的所有文件
2. **读取上游产物**：加载上游阶段产出的文件（用于交叉校验）
3. **读取一致性规则**：加载 `.agent/consistency_rules.yaml`，获取跨视角约束规则
4. **读取跨视角缓存**：加载 `.agent/cross_perspective_cache.yaml`，获取已有的映射关系
5. **执行校验**：
   - **引用完整性**：场景步骤中引用的接口 ID 是否在架构元素 `interfaces.yaml` 中存在
   - **覆盖完整性**：测试用例是否覆盖需求声明的所有场景 ID
   - **格式完整性**：产物文件是否包含必需的 YAML 元数据头部、必需章节
   - **结构完整性**：产物内部引用（如 spec.md 中引用的场景文件）是否有效
6. **生成校验报告**：输出校验结果和不合规项

### 注意事项

- 跨视角一致性是防止信息漂移的核心机制，不允许跳过
- 引用完整性检查为 error 级别，不通过必须修复
- 覆盖完整性检查支持 `allow_partial` 参数（部分覆盖可放行）
- 校验结果同步更新到 `.agent/cross_perspective_cache.yaml`

### 输出要求

- 输出校验报告：校验项列表、通过/不通过/告警、不合规项的详细描述
- 每个不合规项包含：规则 ID、源文件、目标文件、不匹配详情
- 附修复建议
