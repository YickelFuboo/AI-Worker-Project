# 变更总结生成

## 功能描述

为一次变更生成详细的总结文档。获取变更信息 → 分析代码变更（新增文件、修改文件、删除文件）→ 生成时序图 → 生成包/模块依赖图 → 收集测试结果 → 输出完整变更总结文档。

## 适用场景

- 完成一个较大功能开发后，需要输出变更总结
- 需要向团队展示本次变更的全貌
- 发布前需要生成 Release Notes 的技术素材
- 对应参考规则中的 `generate-evaluation`

## 工作方式

### 执行步骤

1. **获取变更信息**：
   - 执行 `git diff` 或 `git log` 获取变更范围
   - 识别新增文件、修改文件、删除文件
   - 读取变更日志（如有）

2. **分析代码变更**：
   - 对新增文件：分析其功能定位、与现有模块的关系
   - 对修改文件：对比修改前后的差异，识别核心变更
   - 对删除文件：确认删除原因和影响

3. **生成功能变化时序图**：
   - 识别本次变更涉及的关键交互流程
   - 以修改前/修改后对比方式，绘制 PlantUML 序列图
   - 标注新增的接口调用、变更的数据流

4. **生成包/模块依赖图**：
   - 分析变更前后的模块依赖关系变化
   - 绘制 PlantUML 组件图，标注新增/删除的依赖边

5. **收集测试结果**：
   - 汇总本次变更相关的测试执行情况
   - 标注测试覆盖率和未覆盖区域

6. **输出变更总结文档**：
   - 按模板组织输出
   - 包含代码变更说明、时序图、依赖图、测试结果

### 注意事项

- 本 skill 适用于较大变更（涉及 3+ 文件或 1+ 新模块），小改动使用 `change-log` 即可
- 时序图聚焦"功能变化"而非"实现细节"
- 依赖图标注模块级别的依赖，不深入到函数级别
- 如果变更包含破坏性 API 变更，在总结中重点标注

### 输出要求

- 变更总结文档：`architectures/changeLogs/{YYYY-MM-DD}-{主题}-summary.md`
- 包含：代码变更清单、功能变化时序图（PlantUML）、模块依赖变化图（PlantUML）、测试结果汇总
