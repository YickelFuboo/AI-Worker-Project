# 读取项目架构（L1 读码 Tool）

## 功能描述

获取代码仓的架构视图：带自然语言说明的模块树、各模块下有哪些文件。用于将用户的自然语言需求映射到子系统/目录，是分层读码策略的第一层（L1）。

## 适用场景

- Agent 收到用户需求后，第一步调用此 Tool 了解项目架构
- 需要知道"哪个模块负责什么功能"
- 在多个候选模块中选择最相关的
- 对应设计文档中的 `read_project_struct` Tool

## 工作方式

### 执行步骤

1. **接收项目标识**：确认查询的 project_id（与构建时一致）
2. **查询图谱架构视图**：从图存储中读取 Project → Folder 的 CONTAINS 层级
3. **返回结构化结果**：
   - 模块树（目录层级）
   - 每个模块的语义摘要（自然语言职责说明）
   - 每个模块下的文件列表（文件名 + 语言）
4. **返回格式**：树形 JSON 或结构化文本

### 注意事项

- 返回内容**不包含**函数体、类实现等源码细节
- 如果图谱未构建，提示用户先执行 `build-graph`
- 返回结果应紧凑，控制在 Agent 上下文消耗可控的范围内
- 模块摘要的语义质量直接影响 Agent 定位准确度

### 输出要求

返回结构：
```
项目: {project_name}
├── {module_1}/ — {模块1职责摘要}
│   ├── {file_1}.{ext} — {文件1职责摘要}
│   └── {file_2}.{ext} — {文件2职责摘要}
├── {module_2}/ — {模块2职责摘要}
│   └── ...
```
