# 接口契约变更分析

## 功能描述

对架构影响域分析中涉及的接口变更进行细化分析。定义接口契约的具体变更内容：新增/修改/废弃的接口字段、参数、返回值、错误码，确保接口契约变更的可追溯性和一致性。

## 所属 Agent

架构设计 Agent

## 适用场景

- 架构影响域分析确定了需要变更的接口
- 需要精确描述接口契约的变更细节
- 需要评估接口变更对调用方和被调方的影响
- 接口变更需要通过兼容性评审

## 工作方式

### 执行步骤

1. **读取架构变更说明**：加载 `architecture_changes/` 下的各元素变更说明
2. **读取接口定义**：加载受影响元素的 `interfaces.yaml`，获取当前接口契约
3. **读取调用关系**：通过依赖关系文件，识别接口的所有调用方
4. **契约变更分析**：逐接口定义变更细节——新增字段、修改字段、废弃字段、返回值变更、错误码变更
5. **兼容性评估**：判断变更是否向后兼容，标注破坏性变更
6. **输出接口变更文档**：追加到对应架构元素的变更说明中

### 注意事项

- 接口契约变更必须精确到字段级别，不允许模糊描述
- 破坏性变更必须列出所有受影响的调用方和迁移方案
- 新增接口必须包含完整的请求/响应格式定义
- 参考 `product_简介.md` 第 9 节的继承性约束：接口变更必须通过兼容性评审

### 输出要求

- 接口变更内容合并到对应元素的变更说明文件中
- 包含接口ID、变更类型、变更前后对比、影响调用方列表
- 破坏性变更需附带迁移方案和兼容性评估
