# 失败根因分析

## 功能描述

对测试执行结果进行分析，分类失败原因。区分产品代码缺陷、测试脚本问题、环境问题、数据问题等失败类型，辅助快速定位和修复。

## 所属 Agent

测试执行 Agent

## 适用场景

- 测试执行完成后，有失败用例需要分析
- 需要快速区分代码缺陷 vs 环境/数据问题
- 需要为测试分析 Agent 提供分类后的失败数据

## 工作方式

### 执行步骤

1. **读取测试报告**：加载测试执行结果（JUnit XML、JSON 等结构化日志）
2. **读取失败详情**：获取失败用例的堆栈信息、断言失败信息、系统日志
3. **读取架构知识**：参考 `architectures/logic_view/elements_tree.yaml` 和相关元素规格，理解失败涉及的系统组件
4. **失败分类**：按以下类别分类：
   - 产品代码缺陷（断言失败、crash、超时）
   - 测试脚本问题（断言逻辑错误、环境配置错误）
   - 环境问题（网络不通、资源不足、依赖服务不可用）
   - 数据问题（测试数据过期、数据格式不匹配）
5. **生成分析报告**：输出分类结果和根因分析

### 注意事项

- 失败分类应基于日志和堆栈的客观证据，不做主观猜测
- 产品代码缺陷需标注可能影响的场景和架构元素
- 环境问题需标注是否需要人工介入

### 输出要求

- 输出失败分类报告，严格按模板格式：`templates/failure_analysis_template.md`
- 包含每个失败用例的类型、可能根因、建议处理方式
- 输出失败统计：按类型分组的失败数量和比例
