# 代码生成

## 功能描述

根据实现设计文档，提取项目现有代码风格和模式，生成风格一致的代码。支持 C/C++、Python 等语言的代码生成，包括函数实现、类定义、数据结构、构建配置更新等。

## 所属 Agent

编码实现 Agent

## 适用场景

- 实现设计文档完成后，需要生成实际代码
- 需要新增函数、类或模块
- 需要修改现有代码以适配新需求
- 需要保持代码风格与项目一致

## 工作方式

### 执行步骤

1. **读取实现设计**：理解 `implementation_design.md` 中的逻辑流、函数签名、数据结构定义
2. **读取现有代码**：阅读待修改文件和周边代码，提取编码风格（命名约定、注释风格、错误处理模式）
3. **读取编码规范**：加载 `knowledge/编码规范/` 中的语言规范和安全规则
4. **读取领域知识**：参考 `knowledge/领域知识/` 中的协议实现模式
5. **代码生成**：按设计文档生成代码，确保风格一致
6. **自检**：对照编码规范进行初步自查

### 注意事项

- 必须提取项目现有模式（错误处理、内存管理、日志风格），保持一致性
- 生成的代码必须遵守 `knowledge/编码规范/安全规则.md` 中的安全编码检查项
- 不生成与设计文档无关的代码，严格按设计实现
- 修改现有代码时，最小化变更范围，不随意重构无关代码
- 涉及外部接口（如 Diameter、S1AP）时，参考领域知识中的协议规范

### 输出要求

- 输出代码 diff（git format-patch 或 unified diff 格式）
- 附合规自检清单（对照编码规范的检查结果）
- 标注生成的代码与设计文档中哪个章节对应
