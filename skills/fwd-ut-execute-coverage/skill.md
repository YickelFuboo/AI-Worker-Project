# 单元测试执行与覆盖率分析

## 功能描述

自动编译并执行单元测试，收集覆盖率数据，分析未覆盖代码分支。生成 UT 执行报告（包含通过率、覆盖率、失败详情），阻断覆盖率不达标的低质量代码。

## 所属 Agent

开发自研 Agent (UT)

## 适用场景

- 单元测试代码生成后，需要编译执行
- 需要分析代码覆盖率（行覆盖、分支覆盖、函数覆盖）
- 需要识别未覆盖的代码路径并给出补充测试建议

## 工作方式

### 执行步骤

1. **读取构建配置**：加载代码仓的 `environment.md`，获取编译工具链和构建脚本
2. **编译测试**：执行测试编译，处理编译错误
3. **执行测试**：运行全部单元测试，收集通过/失败结果
4. **收集覆盖率**：使用 gcov/lcov（C/C++）或 coverage.py（Python）收集覆盖率数据
5. **分析未覆盖路径**：识别覆盖率未达标的代码区域
6. **生成报告**：输出执行报告，包含覆盖率统计和失败分析

### 注意事项

- 覆盖率阈值按项目要求设定（建议行覆盖 ≥80%，分支覆盖 ≥70%）
- 覆盖率不达标应阻断流程，要求补充测试
- 编译失败需分析根因（测试代码问题 vs 被测代码问题）
- 参考 `tests/test_strategy.md` 中的覆盖率目标

### 输出要求

- 输出 UT 执行报告：通过数、失败数、跳过数、通过率
- 输出覆盖率报告：行覆盖率、分支覆盖率、函数覆盖率
- 失败用例详情：失败断言、堆栈信息、可能根因
- 未覆盖代码路径及补充测试建议
