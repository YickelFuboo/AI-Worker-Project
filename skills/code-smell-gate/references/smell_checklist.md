# 代码坏味道检查项与分级阈值

## 必须修复（P0）

### 安全类
| 检查项 | 检测规则 | 阈值 |
|--------|----------|------|
| SQL 注入 | 字符串拼接构造 SQL | 任意 |
| XSS | 未转义的用户输入输出到 HTML | 任意 |
| 硬编码密钥 | API Key / Password 明文 | 任意 |
| 命令注入 | 用户输入拼接进 shell 命令 | 任意 |

### 逻辑类
| 检查项 | 检测规则 | 阈值 |
|--------|----------|------|
| 空指针/空引用 | 未检查 nil/null 就解引用 | 任意 |
| 错误未处理 | 函数返回 error 但调用方未检查 | 任意 |
| 资源泄漏 | 打开的文件/连接未 defer close | 任意 |

## 建议修复（P1）

| 检查项 | 检测规则 | 阈值 |
|--------|----------|------|
| 过长函数 | 函数行数 | > 80 行 |
| 过长参数列表 | 参数个数 | > 5 个 |
| 过深嵌套 | if/for/switch 嵌套层数 | > 4 层 |
| 重复代码 | 相似代码块 | > 6 行 |
| 魔法数字 | 未命名的数字常量 | 非 0/1/-1 |
| 过大文件 | 文件行数 | > 500 行 |
| 未使用的导入 | 导入了但未使用 | 任意 |
| 未使用的变量 | 定义了但未使用 | 任意 |

## 记录记忆（P2）

| 检查项 | 检测规则 |
|--------|----------|
| 注释掉的代码 | 大段注释掉的实现 |
| 不一致的命名风格 | 与项目主流风格不符 |
| TODO/FIXME 未关联 Issue | 无 issue 编号的 TODO |
| 过长一行 | 单行 > 120 字符 |

## 停止原则

以下情况暂停 MR 流程：
- 存在未修复的"必须修复"项
- 修复过程中引入新的"必须修复"项
- 修复后测试失败
- 修改范围超出本次 diff（避免范围蔓延）

## 修复原则

- 修复时只改坏味道相关代码，不做无关重构
- 修复后运行受影响模块的测试
- 不可为了消除告警而弱化逻辑
- 不可为消除告警而引入不安全写法
