# 编写设计文档

## 功能描述

将功能规格说明书转化为技术设计文档。定义模块划分、核心类/函数、数据结构、模块间接口和关键交互流程，为编码实现提供蓝图。

## 适用场景

- 功能规格说明书完成后，需要做技术设计
- 需要确定模块如何划分、接口如何定义
- 需要评估设计决策对现有架构的影响
- 正向流程：需求分析 → 规格说明书 → **设计文档** → 代码

## 工作方式

### 执行步骤

1. **读取 Spec 文档**：理解功能规格中的精确要求
2. **读取现有设计**：阅读 `repo_design.md` 和现有模块设计，了解当前架构
3. **读取代码结构**：使用 `read_project_struct` 和 `read_file_summary` 了解现有代码布局
4. **模块划分**：确定新增/修改哪些模块，各模块职责
5. **接口设计**：定义模块间接口函数签名、参数、返回值
6. **数据设计**：定义核心数据结构、状态机（如有）
7. **流程设计**：绘制关键交互流程（PlantUML 序列图）
8. **DFX 设计**：根据 Spec 的非功能要求，输出安全、性能、可靠性设计方案
9. **输出设计文档**：按模板输出

### 注意事项

- 优先复用现有模块和抽象，不引入不必要的复杂度
- 设计文档聚焦"怎么做"而非"做什么"（那是 Spec 的事）
- 每个设计决策应能追溯到 Spec 中的具体规格项
- 如涉及破坏性变更，必须明确标注影响范围与迁移方案
- 参考 `knowledge/模板库/` 中的 repo_design_template 和 submodel_design_template

### 输出要求

- 仓级设计更新：`architectures/repo_design.md`
- 模块级设计：`architectures/modules/{module_name}/design.md`
- 包含模块划分表、核心类/函数表、数据结构定义、PlantUML 序列图
