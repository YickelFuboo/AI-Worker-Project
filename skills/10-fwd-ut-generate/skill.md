# 单元测试生成

## 功能描述

根据代码变更和实现设计文档，自动生成单元测试代码。覆盖正常路径、异常路径和边界条件，确保测试用例与设计文档中的逻辑流对齐。

## 所属 Agent

开发自研 Agent (UT)

## 适用场景

- 代码生成完成后，需要编写单元测试
- 需要覆盖函数的正常流程、异常流程和边界条件
- 需要保持测试代码风格与项目现有测试一致

## 工作方式

### 执行步骤

1. **读取代码变更**：获取本次生成的代码 diff，提取新增/修改的函数签名
2. **读取实现设计**：理解 `implementation_design.md` 中的逻辑流和关键决策点
3. **读取现有测试**：分析项目中已有的测试代码，提取测试框架、Mock 模式、断言风格
4. **生成测试用例**：按 Given-When-Then 格式生成测试用例，覆盖正常路径、异常路径、边界条件
5. **生成测试代码**：使用项目测试框架生成可编译执行的测试代码
6. **标注测试点**：将测试用例与需求场景 ID 关联

### 注意事项

- 单元测试必须覆盖所有公开函数的所有分支
- 外部依赖（如数据库、网络、文件系统）应使用 Mock 隔离
- 测试代码必须可独立运行，不依赖执行顺序
- 参考 `tests/test_strategy.md` 中的测试策略

### 输出要求

- 输出单元测试代码（.cpp / .py 等）
- 附测试点清单，标注每个测试覆盖的场景 ID
- 标注 Mock 对象和测试桩
