# 构建代码知识图谱

## 功能描述

对存量代码仓执行全量构建，生成代码知识图谱。从源码中提取结构事实（类、函数、模块边界）与依赖事实（调用、导入、继承），为目录、文件、类、函数生成语义摘要，写入图存储，形成代码仓数字孪生。

## 适用场景

- 项目首次启用 AI 辅助开发，需要建立代码仓数字孪生
- 代码仓发生大规模重构后需要重建图谱
- Agent 查询时提示"图谱未构建"
- 子功能二的核心实现

## 工作方式

### 执行步骤

1. **确定项目标识**：读取项目根路径、项目名称，生成唯一 `project_id`
2. **扫描目录树**：遍历项目目录，排除 `.git`、`node_modules`、`vendor`、`build`、`dist` 等非源码目录
3. **创建项目节点**：在图谱中创建 Project 节点，写入项目级描述
4. **文件级处理**（自底向上）：
   - 对每个源文件执行静态分析，提取类、函数、方法等符号
   - 抽取调用关系（CALLS）、导入关系（IMPORTS）、继承关系（INHERITS）
   - 为每个函数/类/文件生成语义摘要（职责说明）
   - 写入节点与边到图谱
5. **目录级聚合**（自底向上）：
   - 聚合子文件、子目录的摘要，生成模块摘要
   - 写入 Folder 节点到图谱
6. **构建完成标记**：记录图谱版本号、构建时间
7. **输出构建报告**：节点数、边数、覆盖文件数、失败文件列表

### 注意事项

- 单文件解析失败不应阻断全项目构建，跳过并记录
- 语义摘要要求简短（1-2句话），偏"职责"而非"实现细节"
- 大文件（>500行）用子符号摘要合成文件摘要，避免单次提炼输入过长
- 多语言项目按文件类型选择分析方式，统一落入同一套概念模型
- 构建过程宜异步执行，大型仓库可能耗时较长

### 输出要求

- 构建报告：`architectures/graph_build_report.md`
- 包含：构建时间、项目标识、节点/边统计、覆盖文件数、失败文件列表
- 图谱数据写入图存储，供后续只读查询
