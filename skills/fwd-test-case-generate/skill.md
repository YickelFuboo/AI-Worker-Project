# 测试用例生成

## 功能描述

根据需求分析结果、场景交互流程和测试点清单，生成或修改测试用例文档。每个测试用例对应一个业务场景，包含前置条件、测试步骤、预期结果和关联场景 ID。

## 所属 Agent

测试设计 Agent

## 适用场景

- 需求分析完成后，需要设计测试用例
- 需要确保测试用例覆盖所有声明的场景
- 需要修改已有测试用例以适配需求变更

## 工作方式

### 执行步骤

1. **读取需求文档**：理解 `requirement.md` 中的场景列表和验收标准
2. **读取测试点**：加载 `requirements/{需求ID}/test_changes/test_points.md` 中的测试点清单
3. **读取场景流程**：加载相关场景交互流程文件（`features/**/SCENARIO_*.md`）
4. **生成测试用例**：按模板输出 `TC_{特性}_{序号}_{描述}.md`，每个用例覆盖一个场景
5. **建立覆盖映射**：确保每个场景 ID 都有对应的测试用例

### 注意事项

- 每个测试用例必须关联到具体的场景 ID，确保可追溯
- 测试用例覆盖正常流程、异常流程和边界条件
- 参考 `knowledge/模板库/测试用例模板.md` 的格式
- 参考 `tests/test_strategy.md` 中的测试策略

### 输出要求

- 输出路径：`tests/test_cases/{特性目录}/TC_{特性缩写}_{序号}_{描述}.md`
- 严格按模板格式输出：`templates/test_case_template.md`
- 包含前置条件、测试步骤（分步描述）、预期结果、关联场景 ID
- YAML 元数据头部包含测试用例 ID、关联场景 ID、优先级
