# 依赖分析：查询函数被调方

## 功能描述

沿代码知识图谱中的 CALLS 边正向查询，找出指定函数调用了哪些其他函数（这个函数调用了谁）。用于理解函数的下游逻辑依赖。

## 适用场景

- 需要理解某个函数的完整执行路径
- 需要评估修改函数内部逻辑时，下游哪些逻辑会受影响
- 调用链追踪
- 对应设计文档中的 `read_function_callees` Tool（P1 规划）

## 工作方式

### 执行步骤

1. **接收参数**：project_id + 文件路径 + 函数名
2. **查询图谱**：从图存储中沿 CALLS 边正向遍历，找到该函数指向的所有被调方
3. **返回结果**：
   - 被调方列表（文件路径 + 函数名）
   - 每个被调方的简要职责摘要
   - 被调方数量统计

### 注意事项

- 图谱中的 CALLS 边由静态分析生成，动态调用可能不完整
- 如果被调方数量很大（> 30），先返回统计数量，Agent 再决定是否全部列出
- 标注被调方是否为外部 API / 第三方库

### 输出要求

```
被调方分析: {function_name}
文件: {file_path}
被调方数量: {n}

被调方列表:
  - {file_path_1} :: {function_name}({params}) — {职责摘要}
  - {file_path_2} :: {function_name}({params}) — {职责摘要}
  ...

外部调用:
  - {external_api_name}
```
