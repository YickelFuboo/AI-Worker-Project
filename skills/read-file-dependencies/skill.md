# 依赖分析：查询文件依赖

## 功能描述

沿代码知识图谱中的 IMPORTS 边查询文件的导入依赖关系。回答"这个文件导入了哪些文件"以及"哪些文件导入了这个文件"。

## 适用场景

- 需要评估文件级变更的影响范围
- 需要理解模块边界和文件间耦合
- 模块拆分/合并前的依赖分析
- 对应设计文档中的 `read_file_dependencies` Tool（P1 规划）

## 工作方式

### 执行步骤

1. **接收参数**：project_id + 文件路径 + 方向（imports / imported_by / both）
2. **查询图谱**：
   - `imports`：沿 IMPORTS 边正向查询，该文件导入了哪些文件
   - `imported_by`：沿 IMPORTS 边反向查询，哪些文件导入了该文件
   - `both`：两个方向都查
3. **返回结果**：
   - 依赖文件列表
   - 依赖关系统计

### 注意事项

- IMPORTS 边仅记录文件间的直接导入关系，不包含传递依赖
- 如果查询 `imported_by` 结果为空，说明该文件可能是叶子节点或被较少引用
- 大规模依赖（> 50 个文件）时建议先返回统计

### 输出要求

```
文件依赖分析: {file_path}

导入 (imports, {n} 个):
  - {file_path_1}
  - {file_path_2}

被导入 (imported_by, {m} 个):
  - {file_path_3}
  - {file_path_4}
```
