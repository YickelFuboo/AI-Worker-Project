# 代码生成

## 功能描述

阅读项目代码，提取模式与约定，然后基于 Spec 和 Design 生成符合项目风格的新代码。通过理解现有实现模式（命名风格、错误处理方式、依赖注入模式等），确保生成的代码与存量代码风格一致。

## 适用场景

- Spec 和 Design 文档已就绪，需要生成新代码
- 需要新增一个模块/类/函数
- 需要生成符合项目规范的样板代码
- 对应参考规则中的 `generate-node`

## 工作方式

### 执行步骤

1. **阅读项目规范**：
   - 阅读 Claude.md / AGENT.md 了解编码规范
   - 阅读 Design 文档了解模块划分和接口约定
2. **提取模式**：
   - 选择 2-3 个与目标模块相似风格的现有模块
   - 提取：命名约定、文件组织方式、错误处理模式、日志格式、测试结构
3. **理解 Spec 要求**：
   - 读取目标功能的 Spec，确认接口契约、数据结构和验收标准
4. **生成代码**：
   - 按项目编码规范生成代码
   - 保持与现有代码一致的风格
5. **生成测试骨架**：
   - 为生成的代码创建对应的测试文件骨架
6. **验证**：
   - 运行现有测试确保未破坏已有功能
   - 运行新测试验证生成代码的正确性

### 注意事项

- 生成的代码必须与项目现有风格一致，不要引入新的编码模式
- 优先复用现有工具函数和抽象
- 如果项目没有 Claude.md，先从现有代码中推断编码规范
- 生成后必须运行测试验证
- 不要生成不必要的文档或注释

### 输出要求

- 生成的源代码文件（按项目约定路径）
- 生成的测试文件骨架
- 简要说明生成的文件清单和遵循的模式来源
