# 依赖分析：查询函数调用方

## 功能描述

沿代码知识图谱中的 CALLS 边反向查询，找出所有调用了指定函数的调用方（谁调用了这个函数）。用于评估修改某个函数时的影响范围。

## 适用场景

- 需要修改某个函数的签名或行为，评估哪些调用方需要适配
- 需要理解某个函数在系统中的使用频率和场景
- 影响面分析的关键步骤
- 对应设计文档中的 `read_function_callers` Tool（P1 规划）

## 工作方式

### 执行步骤

1. **接收参数**：project_id + 文件路径 + 函数名
2. **查询图谱**：从图存储中沿 CALLS 边反向遍历，找到所有指向该函数的节点
3. **返回结果**：
   - 调用方列表（文件路径 + 函数名）
   - 每个调用方的调用上下文（所在文件、所在类）
   - 调用方数量统计

### 注意事项

- 图谱中的 CALLS 边由静态分析生成，动态调用可能不完整
- 如果调用方数量很大（> 50），先返回统计数量，Agent 再决定是否全部列出
- 外部 API 调用方可能跨项目边界，标注"跨项目"

### 输出要求

```
调用方分析: {function_name}
文件: {file_path}
调用方数量: {n}

调用方列表:
  - {file_path_1} :: {class_name}.{method_name}({params})
  - {file_path_2} :: {function_name}({params})
  ...
```
