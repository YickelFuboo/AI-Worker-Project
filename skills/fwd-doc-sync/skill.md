# 文档同步

## 功能描述

代码变更合并后，同步更新相关的设计文档（`spec.md`、`design.md`、模块级设计文档），确保文档与代码长期保持一致。基于代码 diff 识别需要更新的文档范围，最小化同步成本。

## 所属 Agent

编码实现 Agent（代码合并后触发）

## 适用场景

- 代码变更合并到主分支后
- 接口签名、数据结构、关键流程发生变化
- 需要避免文档与代码长期不一致导致后续 Agent 误判

## 工作方式

### 执行步骤

1. **读取代码 diff**：获取本次合并的代码变更（新增/修改/删除的文件、函数、数据结构）
2. **识别影响文档**：根据变更文件路径，映射到相关设计文档：
   - 仓级变更 → `repos/{仓名}/.agent/spec.md`、`design.md`
   - 模块级变更 → `repos/{仓名}/.agent/{模块名}/spec.md`、`design.md`
   - 接口变更 → 同步到 `architectures/logic_view/{元素}/interfaces.yaml`
3. **读取现有文档**：加载待更新文档的当前内容
4. **生成更新**：基于代码 diff 生成文档更新内容：
   - 函数签名变更 → 更新 `design.md` 的函数清单
   - 数据结构变更 → 更新数据结构定义
   - 关键流程变更 → 更新 PlantUML 序列图/状态图
   - 模块职责变更 → 更新 `spec.md` 的业务概述
5. **输出差异摘要**：生成新旧文档的差异摘要，供人工复核

### 注意事项

- 文档同步聚焦"反映代码实际"，不做设计决策变更
- 涉及架构元素接口变更时，需同步触发 `architectures/` 下的元素规格更新
- 大规模重构（如模块拆分/合并）应标注需人工评审的复杂变更
- 小范围代码变更（如 bug 修复、性能优化）一般无需文档同步

### 输出要求

- 更新受影响的 `spec.md` 和 `design.md` 文件
- 输出文档差异摘要：每个文件的更新点清单
- 标注需要人工评审的复杂变更项
