# 代码合规检视

## 功能描述

对生成的代码进行合规检视，对照编码规范和安全规则检查代码质量。包括命名规范、代码风格、安全编码、内存管理、错误处理等方面的静态检查，生成合规检视报告。

## 所属 Agent

编码实现 Agent

## 适用场景

- 代码生成后，需要进行合规检查
- MR 提交前的代码自查
- 需要确保代码符合项目编码规范和安全要求

## 工作方式

### 执行步骤

1. **读取编码规范**：加载 `knowledge/编码规范/` 下的所有规范文件（cpp_规范.md、python_规范.md、安全规则.md、命名规范.md）
2. **读取代码变更**：获取本次生成的代码 diff
3. **逐项检查**：按规范文件中的检查项逐一比对代码
4. **安全专项检查**：重点检查 `knowledge/编码规范/安全规则.md` 中的安全编码项（缓冲区溢出、注入、权限等）
5. **生成报告**：输出合规检视报告，列出违规项、严重级别和建议修复方案

### 注意事项

- 安全规则检查为阻断项，不通过不得提交
- 风格类问题标记为 warning，不阻断但需记录
- 合规检查结果应可追溯到具体规范条款
- 对于电信核心网代码，额外关注内存管理、信号安全和并发安全

### 输出要求

- 输出合规检视报告，包含检查项、结果（通过/不通过/警告）、违规描述和修复建议
- 标注阻断项（error）和告警项（warning）
- 附合规率统计
