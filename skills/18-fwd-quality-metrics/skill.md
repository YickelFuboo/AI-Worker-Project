# 质量度量

## 功能描述

汇总测试执行结果，计算质量度量指标（需求覆盖率、场景覆盖率、代码覆盖率、缺陷密度、通过率），输出质量度量报告。

## 所属 Agent

测试分析 Agent

## 适用场景

- 测试执行完成后，需要评估整体质量
- 需要计算各项覆盖率指标
- 需要为质量报告提供量化数据

## 工作方式

### 执行步骤

1. **读取测试结果**：加载测试执行报告、UT 覆盖率报告、失败分析报告
2. **读取需求文档**：加载 `需求分析.md` 获取声明的场景列表
3. **读取测试用例**：加载 `tests/test_cases/` 下的所有测试用例元数据
4. **读取覆盖映射**：加载 `tests/coverage_mapping.yaml`（如有），获取用例-场景-架构元素映射
5. **计算指标**：
   - 需求场景覆盖率 = 有测试用例的场景数 / 总场景数
   - 代码行覆盖率（来自 UT 执行）
   - 测试通过率 = 通过用例数 / 总用例数
   - 缺陷密度 = 发现缺陷数 / 代码变更行数
6. **生成质量报告**：输出质量度量报告

### 注意事项

- 需求场景覆盖率 ≥95% 为目标值（参见产品简介中的关键成功指标）
- 各项指标的计算公式和阈值参考 `tests/test_strategy.md`
- 缺失的覆盖数据应明确标注

### 输出要求

- 输出质量度量报告，严格按模板格式：`templates/quality_report_template.md`
- 包含各项指标的计算值和目标值对比
- 输出覆盖缺口清单：未覆盖的场景 ID 和原因
- 输出质量趋势（如有历史数据）
