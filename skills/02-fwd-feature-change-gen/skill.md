# 业务变更说明生成

## 功能描述

基于结构化需求分析结果，生成业务视角的变更说明文档。明确受影响的一级特性、子特性和场景，描述每个场景的变更内容（新增/修改/删除），生成 `feature_changes/` 目录下的变更文件。

## 所属 Agent

需求分析 Agent

## 适用场景

- 需求分析完成后，需要明确业务视角的变更范围
- 需要确定新增或修改的场景及其交互流程变更
- 为后续架构影响域分析提供业务侧输入

## 工作方式

### 执行步骤

1. **读取需求文档**：解析 `requirement.md` 中的功能概述、场景列表和验收标准
2. **读取特性树**：加载 `features/features_tree.yaml` 和相关二级索引文件，定位受影响的一级特性和子特性
3. **分析场景影响**：逐场景判断是新增、修改还是删除，记录变更原因
4. **生成变更文件**：按模板输出到 `requirements/{需求ID}/feature_changes/{子特性目录}/` 下，命名格式 `SR-{序号}-{场景描述}_变更说明.md`
5. **更新继承性报告**：标注变更对存量场景的影响

### 注意事项

- 变更说明必须精确引用场景 ID（如 SCENARIO_001），确保可追溯
- 场景间如有依赖关系，需在变更说明中明确顺序
- 参考 `knowledge/模板库/业务变更说明模板.md` 的格式

### 输出要求

- 输出路径：`requirements/{需求ID}/feature_changes/{子特性目录}/SR-{序号}-{描述}_变更说明.md`
- 严格按模板格式输出：`templates/feature_change_template.md`
- 包含场景ID、变更类型、变更前/后描述、交互流程差异
- YAML 元数据头部包含需求ID、子特性ID、变更类型
