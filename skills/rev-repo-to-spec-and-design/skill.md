# 设计逻辑还原

## 功能描述

为复杂模块从源代码反向生成逻辑设计说明。提取模块的类/函数结构、数据结构、状态机和关键流程，生成模块级的 `spec.md` 和 `design.md`（含 PlantUML 图）。

## 所属 Agent

实现逆向 Agent

## 适用场景

- 复杂模块缺少设计文档，需要从代码还原设计
- 模块代码变更后需要更新设计文档
- 存量反向分析流程的第三步（业务→架构→实现）

## 工作方式

### 执行步骤

**模式判定**：检查目标产物 `repos/{仓名}/.agent/spec.md` 与 `design.md` 是否存在且 YAML 头部有 `last_modified`。两者均存在 → 进入增量更新模式；任一不存在 → 进入首次生成模式。

#### 首次生成模式

1. **读取模块源码**：扫描指定模块目录下的全部源代码
2. **读取仓级设计**：加载仓级 `design.md`，理解模块在仓库中的位置和与其他模块的关系
3. **读取领域知识**：参考 `knowledge/领域知识/` 中的协议规范和常见模式
4. **代码分析**：
   - 提取类/函数清单和签名
   - 识别核心数据结构
   - 提取状态机实现（switch-case、状态表、枚举+转换函数）
   - 追踪关键函数的调用链
5. **逻辑还原**：从代码推导设计意图，绘制 PlantUML 状态图和序列图
6. **生成设计文档**：
   - 模块 `spec.md`：定位、职责、关联业务/架构概念
   - 模块 `design.md`：类/函数说明、数据结构、状态机图、序列图

#### 增量更新模式

1. **解析时间锚**：读取 `spec.md` 与 `design.md` YAML 头部 `last_modified`（若两者不一致，取较早者作为锚点，保证不漏变更）
2. **定位锚点提交**：`git log --before="<last_modified>" -1 --format=%H` 得到上次刷新时的最近提交
3. **变更探测**：`git diff <锚点提交>..HEAD --name-status` 得到新增/修改/删除文件清单
4. **变更分类与章节映射**：
   - `go.mod` / 依赖声明文件变更 → spec.md 技术栈表、外部依赖表；design.md 模块间接口
   - 入口文件（cmd/、main）变更 → spec.md 功能规格；design.md 关键跨模块流程
   - 核心数据结构文件（如 `context/amf_ue.go`）变更 → design.md 核心数据对象；spec.md 业务功能
   - 状态机文件变更 → design.md 状态机 PlantUML 图
   - 新增模块/目录 → spec.md 目录概览、模块清单；design.md 模块划分、模块详细设计
   - 删除模块 → 同步移除对应章节，不得遗留
   - 仅注释/格式变更 → 跳过
5. **刷新决策**：
   - 无变更 → 不刷新，仅更新 `last_modified` 为检查时间并记录"无变更"
   - 局部变更 → 仅重读受影响文件 + 直接依赖，刷新对应章节，保留未受影响章节
   - 结构性变更（新增模块、状态机重构、对外接口大改）→ 触发首次生成式全量刷新
6. **刷新执行**：按模板比对现有内容，合并更新；PlantUML 图若调用链未变可保留，否则重绘
7. **更新时间戳与报告**：更新 `last_modified`，输出差异摘要（变更文件清单、刷新章节、跳过章节与原因、置信度变化）

### 注意事项

- 仅当模块源码文件数 > 800 时输出独立的模块级设计文档；否则信息融合到仓级文档
- PlantUML 图必须准确反映代码中的实际状态转换和调用关系
- 参考 skill 自带的逆向扩展模板（含置信度、代码位置约定）
- 增量模式下，若仓内有 rebase/squash 导致 `last_modified` 早于实际最近提交，需以锚点提交为准
- 删除文件若对应废弃模块/特性，需从模块清单、目录概览中同步移除
- 增量刷新不应降低置信度；若变更引入不确定项，置信度维持原值并在报告中标注新增不确定项

### 输出要求

- 模块级文档（模块源码文件数 > 800）：`repos/{仓名}/.agent/{模块名}/spec.md` 和 `design.md`
  - `spec.md` 严格按模板：`templates/module_spec_template.md`
  - `design.md` 严格按模板：`templates/module_design_template.md`
- 仓级文档（小模块，信息融合到仓级）：`repos/{仓名}/.agent/spec.md` 和 `design.md`
  - `spec.md` 严格按模板：`templates/repo_spec_template.md`
  - `design.md` 严格按模板：`templates/repo_design_template.md`
- 包含 PlantUML 状态图（如有状态机）和序列图（关键流程）
- 不再单设"逆向来源"章节；置信度写入 YAML 头部，不确定项与人工复核点写入本次生成/更新的差异摘要（一次性产物，不持久化到 md 正文）
- 如为更新已有文档，输出差异摘要
