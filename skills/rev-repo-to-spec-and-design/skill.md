# 设计逻辑还原

## 功能描述

为复杂模块从源代码反向生成逻辑设计说明。提取模块的类/函数结构、数据结构、状态机和关键流程，生成模块级的 `spec.md` 和 `design.md`（含 PlantUML 图）。

## 所属 Agent

实现逆向 Agent

## 适用场景

- 复杂模块缺少设计文档，需要从代码还原设计
- 模块代码变更后需要更新设计文档
- 存量反向分析流程的第三步（业务→架构→实现）

## 工作方式

### 执行步骤

1. **读取模块源码**：扫描指定模块目录下的全部源代码
2. **读取仓级设计**：加载仓级 `design.md`，理解模块在仓库中的位置和与其他模块的关系
3. **读取领域知识**：参考 `knowledge/领域知识/` 中的协议规范和常见模式
4. **代码分析**：
   - 提取类/函数清单和签名
   - 识别核心数据结构
   - 提取状态机实现（switch-case、状态表、枚举+转换函数）
   - 追踪关键函数的调用链
5. **逻辑还原**：从代码推导设计意图，绘制 PlantUML 状态图和序列图
6. **生成设计文档**：
   - 模块 `spec.md`：定位、职责、关联业务/架构概念
   - 模块 `design.md`：类/函数说明、数据结构、状态机图、序列图

### 注意事项

- 仅当模块源码文件数 > 800 时输出独立的模块级设计文档；否则信息融合到仓级文档
- PlantUML 图必须准确反映代码中的实际状态转换和调用关系
- 参考 skill 自带的逆向扩展模板（含置信度、代码位置、逆向来源章节）

### 输出要求

- 模块级文档（模块源码文件数 > 800）：`repos/{仓名}/.agent/{模块名}/spec.md` 和 `design.md`
  - `spec.md` 严格按模板：`templates/module_spec_template.md`
  - `design.md` 严格按模板：`templates/module_design_template.md`
- 仓级文档（小模块，信息融合到仓级）：`repos/{仓名}/.agent/spec.md` 和 `design.md`
  - `spec.md` 严格按模板：`templates/repo_spec_template.md`
  - `design.md` 严格按模板：`templates/repo_design_template.md`
- 包含 PlantUML 状态图（如有状态机）和序列图（关键流程）
- 必须填写"逆向来源"章节：源码位置、置信度评估依据、不确定项、人工复核点
- 如为更新已有文档，输出差异摘要
