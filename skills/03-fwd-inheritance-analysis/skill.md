# 需求继承性分析

## 功能描述

分析新需求对存量功能的影响，评估需求变更是否会破坏已有特性子场景的接口契约与信令流程。生成继承性分析报告，标记潜在的兼容性风险点和回归测试范围。

## 所属 Agent

需求分析 Agent

## 适用场景

- 需求涉及修改已有子特性或场景时
- 需要评估新需求与存量特性的兼容性
- 需要确定回归测试的范围和重点
- 版本演进中需遵守"不破坏已有特性"的继承性约束

## 工作方式

### 执行步骤

1. **读取需求文档**：理解新需求的功能范围和场景列表
2. **读取特性树**：加载 `features/` 下所有相关子特性的 `spec.md` 和 `arch_ref.yaml`
3. **读取架构关联**：分析受影响场景涉及的架构元素和接口
4. **交叉比对**：识别新需求与存量场景的功能重叠、接口共用、数据冲突点
5. **风险评估**：按高/中/低分级标记每个兼容性风险
6. **生成报告**：输出继承性分析报告，包含风险项、影响范围和建议的回归测试范围

### 注意事项

- 继承性分析是版本演进的关键质量保障环节，不可跳过
- 高风险项（如接口契约变更）必须标记为人工审核点
- 分析结果应直接指导测试设计 Agent 的回归测试范围
- 参考 `产品简介.md` 第 9 节的版本演进与约束

### 输出要求

- 输出路径：`requirements/{需求ID}/inheritance_report.md`
- 包含风险矩阵（风险项、级别、影响范围、缓解建议）
- 包含建议的回归测试场景 ID 列表
