# 环境拓扑建模

## 功能描述

解析部署架构文档，构建测试环境的拓扑模型。包括节点分布、网络连接、资源规格、服务依赖关系，用于生成测试环境部署脚本和配置。

## 所属 Agent

测试执行 Agent

## 适用场景

- 测试执行前，需要搭建测试环境
- 需要根据部署拓扑生成环境配置
- 需要验证测试环境是否符合部署规格

## 工作方式

### 执行步骤

1. **读取部署文档**：加载 `architectures/deployment_view/deployment.md`，解析节点、网络、资源规格
2. **读取环境配置**：加载 `architectures/deployment_view/environment.md`，获取编译工具链和运行时依赖
3. **构建拓扑模型**：生成节点间网络连接图、服务依赖关系图
4. **生成部署配置**：输出测试环境的部署脚本和配置文件（如 docker-compose、k8s manifests）
5. **环境校验**：检查部署配置与实际环境的匹配度

### 注意事项

- 拓扑模型必须精确反映部署文档中的节点规格和网络划分
- 测试环境的资源规格可以低于生产环境，但拓扑结构必须一致
- 参考 `architectures/deployment_view/deployment.md` 中的 PlantUML 部署图

### 输出要求

- 输出拓扑模型（YAML/JSON 格式，节点列表+连接关系）
- 输出部署脚本和配置文件
- 输出环境校验清单
