# 版本打包

## 功能描述

执行多仓编译并生成版本包。包括源码编译、链接、二进制打包、版本签名和版本包校验。处理编译失败并生成编译日志。

## 所属 Agent

编译构建 Agent

## 适用场景

- 所有代码变更完成后，需要编译并打包
- 需要生成可部署的版本包
- 编译失败时需要分析并报告

## 工作方式

### 执行步骤

1. **读取编译计划**：获取多仓依赖解析的编译顺序
2. **执行编译**：按顺序逐仓编译，捕获编译输出和错误
3. **处理编译失败**：分析编译错误类型（语法错误、链接错误、依赖缺失），生成失败报告
4. **版本打包**：编译成功后，按部署规范打包二进制、配置、脚本
5. **版本签名**：对版本包进行数字签名
6. **生成编译日志**：输出完整编译日志和版本包清单

### 注意事项

- 编译失败需区分代码问题 vs 环境问题
- 版本包必须包含版本号、构建时间、commit 哈希
- 参考 `architectures/deployment_view/deployment.md` 中的部署包结构

### 输出要求

- 输出版本包（tar.gz / rpm / docker image 等格式）
- 输出编译日志（stdout + stderr）
- 输出版本包清单（包含文件列表、版本号、签名）
