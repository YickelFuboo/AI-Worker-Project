# 多仓依赖解析

## 功能描述

分析多代码仓之间的编译依赖关系，确定编译顺序和依赖版本。处理跨仓依赖冲突，生成编译依赖图，确保所有仓能正确编译。

## 所属 Agent

编译构建 Agent

## 适用场景

- 多仓项目需要确定编译顺序
- 跨仓依赖变更需要评估影响范围
- 编译失败时需要分析依赖问题

## 工作方式

### 执行步骤

1. **读取元素树**：加载 `architectures/logic_view/elements_tree.yaml`，获取代码仓间的逻辑依赖关系
2. **读取构建配置**：加载各代码仓的 `environment.md`，提取编译工具链、依赖库版本
3. **解析依赖图**：构建有向无环图（DAG），确定编译拓扑顺序
4. **依赖版本分析**：检查跨仓依赖的版本一致性，标记冲突
5. **生成编译计划**：输出编译顺序和每个仓的编译命令

### 注意事项

- 依赖图必须是无环的，发现循环依赖立即报错
- 跨仓 API 版本不兼容视为阻断项
- 参考 `architectures/deployment_view/environment.md` 中的全局编译环境配置

### 输出要求

- 输出依赖 DAG（PlantUML 或 Mermaid 格式）
- 输出编译顺序清单（拓扑排序结果）
- 标注版本冲突和缺失依赖
