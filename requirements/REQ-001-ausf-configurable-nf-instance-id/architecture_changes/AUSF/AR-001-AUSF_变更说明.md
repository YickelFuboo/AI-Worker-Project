# 架构元素变更说明：AUSF

## 1. 变更概述
| 项目 | 内容 |
|------|------|
| 关联需求 | REQ-001-ausf-configurable-nf-instance-id |
| 架构元素 | ausf AUSF |
| 变更类型 | 修改 |
| 变更原因 | AUSF 需要支持可选的稳定 NF Instance ID 配置，并在未配置时保持自动生成行为，在非法配置时拒绝启动。 |
| 来源场景 | SCENARIO_001 / FC-001 / SI-001；SCENARIO_002 / FC-002 / SI-002；SCENARIO_003 / FC-003 / SI-003 |

## 2. 元素职责影响
| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 元素定位 | AUSF 是 5G 核心网控制面的鉴权服务器网元，承担 UE 主鉴权、鉴权结果上报、NRF 注册与发现等职责。 | 元素定位保持不变，仍为鉴权服务器网元；启动身份能力扩展为支持可选稳定 NF Instance ID 配置。 |
| 主要职责 | AUSFContext 全局上下文维护 NF 实例信息；启动期通过 NRF 注册 NF 实例；未配置稳定实例标识时按既有行为生成 NF Instance ID。 | AUSFContext 和启动流程需要在 NF 实例信息建立前处理可选 nfInstanceId：合法 UUID v4 配置值优先，未配置时保持自动生成，非法或非 UUID v4 时启动失败并提示配置错误。 |
| 业务能力 | 现有能力包含 UE 主鉴权编排、5G-AKA / EAP-AKA' 算法执行、鉴权结果上报、NRF 注册与发现、OAuth2 接入控制和可观测性出口。 | 新增“稳定 NF Instance ID 配置”架构能力；不改变 UE 鉴权编排、鉴权算法、AMF/UDM 业务交互语义或 Nausf 鉴权服务能力。 |

## 7. 接口契约分析（05）

### 7.1 IC-001 NRF NF 实例注册相关接口
| 项目 | 内容 |
|------|------|
| 来源候选影响 | CII-001 |
| 关联架构影响 | AI-001 |
| 接口影响类型 | modify |
| 涉及接口 | NRF RegisterNFInstance / NF 实例注册相关接口 |
| 来源场景 | SCENARIO_001；SCENARIO_002；SCENARIO_003 |
| 影响调用方 | AUSF、NRF、外部读取 NRF 注册信息的运维或检查系统 |
| 破坏性变更 | false |

#### 契约变化摘要
- 请求变化：不新增 NRF 注册接口字段；NF Instance ID 的可观察值来源从仅自动生成扩展为合法配置值优先、未配置时自动生成。
- 响应变化：NRF 注册响应契约无变化。
- 错误语义变化：非法或非 UUID v4 的 `nfInstanceId` 在 AUSF 启动阶段失败，不形成 NRF 注册接口的运行期错误码变化。
- 兼容性结论：未配置 `nfInstanceId` 的存量部署仍使用自动生成值，不破坏既有 NRF 注册调用；配置合法值后，外部通过 NRF 注册信息观察到的 NF Instance ID 可保持稳定。

#### 契约变化摘要
- 请求变化：外部观察入口未确认，暂无法判断请求契约是否变化。
- 响应变化：外部观察入口未确认，暂无法判断响应或输出契约是否变化。
- 错误语义变化：非法配置属于 AUSF 启动失败路径；是否通过日志、健康检查、指标或其它出口体现，仍需后续确认。
- 兼容性结论：该影响保留为 `unknown`，不得在后续阶段隐式假设为确定接口变更或确定无变更。
