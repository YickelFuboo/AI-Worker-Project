# 5GC 架构与协议

> 适用规范：TS 23.501（架构） / TS 23.502（流程） / TS 29.500（SBA Stage3） / TS 38.413（NGAP） / TS 29.244（PFCP）。
> 本文档把架构、参考点、协议栈、典型流程放在一起，作为查漏补缺的入口。详细条款号见各小节末尾。

## 1. 总体架构（SBA, Service-Based Architecture）

5GC 放弃了 4G EPC 的"网元间点对点接口"模型，改用**服务化架构**：每个 Network Function (NF) 把自己的能力以 HTTP/2 REST 服务的形式暴露出来，其它 NF 通过服务总线（SBI）调用。点对点接口（如 N1–N4、N6）只在无法服务化时保留（主要是 UE-RAN-UPF 链路）。

```
                 ┌─────────────── SBI (HTTP/2 + JSON) ───────────────┐
   UE ── N1 ── AMF  ──┐  │  ┌── SMF ── N4 ── UPF ── N6 ── DN
                     │  │  │
   (R)AN ── N2 ── AMF │  │  │
                     │  │  │
                     └──┼──┼──> NRF / UDM / UDR / AUSF / PCF / NSSF / NEF / CHF / BSF
                        │  │     ↑ 所有 NF 都通过 SBI 互访，NRF 做服务发现
                        └──┘
```

关键特征：
- **控制面 / 用户面分离**：CP（AMF/SMF/...）与 UP（UPF）独立部署、独立扩缩容。
- **NF Discovery**：通过 NRF (Network Repository Function) 注册与发现，TS 23.501 §6.2.6。
- **NF Set**：同质 NF 可组成 Set，无状态、可负载分担；状态外置到 UDSF（UDM/UDR 也承担部分）。
- **Network Slicing**：用 S-NSSAI（Slice/Service Type + Slice Differentiator）标识切片，TS 23.501 §5.15。
- **漫游**：home-routed 与 local breakout 两种模式，hPLMN/vPLMN 各部署一套 NF，TS 23.501 §5.16。

参考：TS 23.501 §4（System architecture）、§6（High-level functions）。

## 4. 协议栈

### 4.1 控制面 SBI（Nxx 接口）

```
应用层：  3GPP NF Service API (RESTful)
表示层：  JSON (3GPP schemas, TS 29.571 数据类型)
传输层：  HTTP/2 (TLS 强制；OAuth2 可选)
```

- TS 29.500 §4：SBI 通用原则，HTTP/2 必选、TLS 1.2/1.3、JSON 编码。
- TS 29.501：5GC SBI Stage 3 详细原则。
- 服务操作命名：`N<XX>_<ServiceName>_<Operation>`，如 `Nsmf_PDUSession_CreateSMContext`。
- 资源命名：`/nf-instance/{nfInstanceID}`、`/sm-contexts/{smContextRef}` 等。
- 错误模型：`ProblemDetails` RFC 7807 扩展，TS 29.500 §4.8。

### 4.2 NAS (Non-Access-Stratum)

UE 与 AMF 之间的层，TS 24.501：

```
应用层：  NAS (5GMM / 5GSM)
传输层：  RRC → PDCP → RLC → MAC → PHY (over Uu)
                 或 IPSec (over N3IWF/TNGF untrusted non-3GPP)
```

- 5GMM：移动性管理（注册/去注册/服务请求）。
- 5GSM：会话管理（PDU 会话建立/修改/释放）。
- 安全：NAS 加密 (NEA0–NEA3) + 完整性 (NIA0–NIA3)，TS 33.501。

### 4.3 NGAP (N2)

(R)AN ↔ AMF 之间，TS 38.413：

```
应用层：  NGAP
传输层：  SCTP (stream)
```

关键流程：NG Setup、Initial UE Message、PDU Session Resource Setup/Modify/Release、Path Switch Request、Handover、UE Context Release、Paging。

### 4.4 GTP-U (N3 / N9)

(R)AN ↔ UPF / UPF ↔ UPF，TS 29.281：

```
应用层：  GTP-U
传输层：  UDP/IP
```

N3 隧道用 TEID 区分 PDU 会话；QoS Flow 用 QFI 在 GTP-U 扩展头中携带。

### 4.5 PFCP (N4)

SMF ↔ UPF，TS 29.244：

```
应用层：  PFCP
传输层：  UDP/IP
```

PFCP 建立 Session，关联 PDR（检测规则）、FAR（转发规则）、QER（QoS 规则）、URR（使用上报规则）、BAR（缓冲规则）。go-upf 用 `internal/pfcp` 解析这些 IE。

## 5. QoS 模型

5GC 用 **5QI (5G QoS Identifier)** 标识 QoS Flow，TS 23.501 §5.7：

- **GBR QoS Flow**：保证比特率，对应 5QI 1–4, 65, 66, 79。
- **Non-GBR QoS Flow**：默认 5QI 9 (best effort)、5QI 5 (IMS 信令) 等。
- **参数**：5QI、ARP (Allocation/Retention Priority)、GBR/MBR (仅 GBR)、Session-AMBR (Non-GBR 会话聚合)。
- **反射 QoS**：UE 通过下行包的 RQI 反推上行 QoS 规则，无需显式下发。

QoS Flow 与 DRB 的映射由 (R)AN 决定。SMF 通过 N4 把 QER 下发到 UPF，UPF 按 QFI 标记 GTP-U 扩展头。

free5GC 实现状态：control-plane only 5QI/ARP/GBR/MBR + Session-AMBR（v3.3.0+）。

## 6. 典型流程（关键步骤定位）

### 6.1 初始注册（Registration, TS 23.502 §4.2.2）

1. UE → RAN → AMF：`Registration Request` (InitialRegistration, SUCI/SUPI)。
2. AMF → AUSF → UDM：请求鉴权向量；AUSF 走 5G-AKA 或 EAP-AKA'。
3. AMF ↔ UE：NAS Security Mode Command / Complete。
4. AMF → UDM：`Nudm_UECM_Registration`（注册当前 AMF），`Nudm_SDM_Get`（取签约数据）。
5. AMF → PCF：`Npcf_AMPolicyControl_Create`（建立 AM 策略关联）。
6. AMF → UE：`Registration Accept`（含 5G-GUTI、TAI List、Allowed NSSAI）。

### 6.2 PDU 会话建立（TS 23.502 §4.3.2）

1. UE → AMF：`PDU Session Establishment Request` (NAS 5GSM)。
2. AMF → SMF：`Nsmf_PDUSession_CreateSMContext`（带 DNN、S-NSSAI、SUPI）。
3. SMF → UDM：`Nudm_SDM_Get` 取 DNN 签约；SMF → PCF `Npcf_SMPolicyControl_Create`。
4. SMF → UPF（N4）：PFCP Session Establishment（含 PDR/FAR/QER/URR）。
5. SMF → AMF：`Nsmf_PDUSession_UpdateSMContext` 返回 N2 SM Info（PDU Session Resource Setup）。
6. AMF → RAN：`PDU Session Resource Setup` (NGAP)。
7. RAN ↔ UE：RRC 重配建立 DRB。
8. RAN → AMF → SMF：`PDU Session Resource Setup Response`（含 N3 TEID）。
9. SMF → UPF：`PFCP Session Modification` 更新 N3 TEID。

### 6.3 服务请求（Service Request, TS 23.502 §4.2.3）

- UE 触发：上行数据或信令需要，发 `Service Request` 给 AMF；AMF 触发 AN 资源建立。
- 网络触发：下行数据到达 UPF，UPF 通知 SMF，SMF → AMF `Namf_Communication_N1N2MessageTransfer`（Paging）。

### 6.4 切换

- **N2 Handover**（TS 23.502 §4.9.1）：源 AMF → 目标 AMF（同 PLMN 跨 AMF 时走 N14）；源 SMF 协调 UPF 切换 N3 路径。free5GC 不支持 indirect mode。
- **Xn Handover**（TS 23.502 §4.9.2）：RAN 间直接 Xn 切换，事后通知 AMF/SMF 更新路径。

### 6.5 去注册（Deregistration, TS 23.502 §4.2.4）

UE-initiated 与网络-initiated 两种变体；释放 SM Context、N4 Session、AM Policy 关联。

## 7. 切片（Network Slicing）

- **S-NSSAI = SST (8 bit) + SD (24 bit)**，TS 23.501 §5.15.2。
- 配置 NSSAI、Allowed NSSAI、Requested NSSAI 三态。
- AMF 通过 NSSF 选择支持目标切片的 SMF/UPF；NSSF 也返回 Allowed NSSAI。
- free5GC 支持 Multiple Slices + DNN，可基于 S-NSSAI 选 UPF。

## 8. 安全（TS 33.501）

- 主鉴权：5G-AKA（向 AMF 隐藏鉴权结果，防伪基站）或 EAP-AKA'。
- 密钥层次：K (永久) → KAUSF → KSEAF → KAMF → KNASenc / KNASint / KN3IWF / KgNB。
- NAS 安全：NEA0–NEA3 加密、NIA0–NIA3 完整性。
- UP 安全：用户面加密 + 完整性（可选），由 SMF 通过 N4 下发到 UPF。
- SBI 安全：TLS + OAuth2（free5GC 在 `Enable OAuth2 on SBI` 配置后启用）。

## 9. 计费（TS 32.240 / 32.255）

- **离线计费**：CHF 接收 CDR（Charging Data Record），经 CGF (Charging Gateway Function) 转给 Billing Domain。
- **在线计费**：CHF 实时扣费，含 ABMF (Account Balance Management Function) + RF (Rating Function)。
- **Converged Charging**：SMF 通过 N40 (Nchf) 上报 Usage Report（流级 / 会话级）。
- free5GC：内置 CHF 含 ABMF/RF/CGF，webconsole 充当 Billing Domain 前端。
